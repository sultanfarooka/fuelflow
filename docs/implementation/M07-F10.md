## Summary

M07-F07 shipped the sidebar chrome but left the nav item catalog incomplete:
only 8 items (Organization, Station, Shifts, Inventory, Finance, Reports,
Setup, Settings). Six more modules need nav entries — Fuel Pricing (M06),
Nozzle Operations (M03), Credit Customers (M15), Users & Access (M01),
Staff & Payroll (M13), Lubricants / Oil Shop (M09) — plus group labels,

an unbuilt module renders a shared `<UnderDevelopment />` page; clicking a
plan-gated module on a Starter plan renders a shared `<UpgradePrompt />`.

## Decisions (from planning interview)

1. **Custom User approximation.** M01-F06 permission grants are not in the
   auth store yet. For M07-F10, Custom users see any nav item whose `roles`
   array includes `ROLES.Custom`. Admin-group items never include Custom.
   M01-F06 will add per-item filtering on top.

2. **Plan tier detection.** `SubscriptionInfo.planName` is the available
   signal. Plan-gated routes check:
   `const isProPlan = subscription?.planName != null && subscription.planName.toLowerCase() !== 'starter';`
   This is an approximation; M11-F06 will own the authoritative plan-feature
   gating. If `subscription` is null (no active plan), treat as Starter
   (show the upgrade prompt).

3. **Nav items always visible — plan gate is a page-level concern.** Plan-gated
   items (Staff, Lubricants) are always visible in the sidebar to Owner;
   clicking them renders `<UpgradePrompt />` (Starter) or `<UnderDevelopment />`
   (Pro+). They are not hidden from the nav.

4. **Separate `UnderDevelopment` component from `ComingSoon`.** `ComingSoon`
   stays backward-compatible for existing M07-F07 stubs. New M07-F10 stubs use
   the richer `<UnderDevelopment moduleName description icon />` component.

5. **Groups rendered in sidebar via label.** `NavItemConfig` gets a `group`
   field; `app-sidebar.tsx` groups items by `group` and renders a
   `SidebarGroupLabel` per group that has at least one visible item.

6. **Dashboard ("Organization") item rename + group.** Currently key
   `"organization"`, labelKey `"nav.organization"`. Rename labelKey to
   `"nav.dashboard"` (already exists in en.json) and place in `operations`
   group. Key stays `"organization"` to avoid breaking the active-route logic
   that special-cases it.

7. **Settings Settings (M08) visible to Manager.** The existing nav entry has
   `roles: [ROLES.Owner]`. The catalog says Manager + Owner. Update Settings
   entry and the `settings/route.tsx` `beforeLoad` guard to allow both.

8. **No admin-group layout route needed.** Admin sub-routes
   (`/admin/users`, `/admin/staff`, `/admin/lubricants`) use flat files
   `station.$stationId.admin.users.tsx` etc. inside `routes/dashboard/`.
   TanStack Router resolves them inside the dashboard `route.tsx` layout
   automatically; no `admin/route.tsx` is needed.
