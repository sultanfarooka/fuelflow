## Summary

Adds a collapsible **"Station Management"** sidebar group (Owner + Manager only)
with four child nav items: Fuel Types, Fuel Tanks, Fuel Pricing, and Nozzles.
Each child renders the shared `<UnderDevelopment />` placeholder until its
backing feature ships. The group auto-expands when any child route is active;
it is absent for Custom Users. Fuel Pricing moves from the Commercial nav group
to this hub (M08-F07-R05). Three new placeholder routes are added under
`/dashboard/station/:stationId/manage/` for Fuel Types, Tanks, and Nozzles;
the existing `/pricing` route is reused for Fuel Pricing (its nav item moves
group, the route file stays).

## Acceptance criteria (the test plan)

From `MODULES.md`:

- **AC1** Given an Owner or Manager on the station dashboard, when they view
  the sidebar, then a "Station Management" group is visible containing exactly:
  Fuel Types, Fuel Tanks, Fuel Pricing, Nozzles.
- **AC2** Given a Custom User on the station dashboard, when they view the
  sidebar, then no "Station Management" group or its child items appear.
- **AC3** Given any authenticated user, when they view the sidebar Commercial
  group, then "Fuel Pricing" is absent from it.
- **AC4** Given an Owner navigates to any child under Station Management, when
  the sidebar renders, then the Station Management group is expanded (not
  collapsed).
- **AC5** Given an Owner clicks any Station Management child item whose backing
  feature is not yet built, then the shared "Under Development" placeholder
  renders.

### Phase 4 — Collapsible sidebar group (app-sidebar.tsx)

The `"station-management"` group requires a different render from the flat
groups. Extend `app-sidebar.tsx`:

1. **Add imports:**
   ```ts
   import { Collapsible, CollapsibleContent, CollapsibleTrigger } from "radix-ui"
   import { IconChevronDown, IconSettings2 } from "@tabler/icons-react"
   import {
     SidebarMenuSub,
     SidebarMenuSubButton,
     SidebarMenuSubItem,
   } from "@/components/ui/sidebar"
   ```

2. **Derive auto-expand state** — fully controlled: open whenever any child is
   active (M08-F07-R04):
   ```ts
   const stationMgmtItems = items.filter((i) => i.group === "station-management")
   const isStationMgmtOpen = stationMgmtItems.some((item) => {
     const resolved = resolvePath(item.to, item.params)
     return isActive(item.key, resolved, pathname)
   })
   ```
   When no child is active, the group renders collapsed. The user can manually
   toggle, but navigating to a child always forces it open. Implement with
   `Collapsible open={isStationMgmtOpen || manualOpen} onOpenChange={setManualOpen}`;
   on navigation to a child, `isStationMgmtOpen` becomes `true` which overrides
   `manualOpen`. Use `useState(false)` for `manualOpen`.

3. **Special-case render for `"station-management"` group** — inside the
   `NAV_GROUPS.map(...)` loop, add a branch before the existing group render:
   ```tsx
   if (group === "station-management") {
     if (stationMgmtItems.length === 0) return null
     return (
       <SidebarGroup key={group}>
         <SidebarGroupLabel>{t("nav.groups.stationManagement")}</SidebarGroupLabel>
         <SidebarGroupContent>
           <SidebarMenu>
             <SidebarMenuItem>
               <Collapsible
                 open={isStationMgmtOpen || manualOpen}
                 onOpenChange={setManualOpen}
                 className="group/station-mgmt"
               >
                 <CollapsibleTrigger asChild>
                   <SidebarMenuButton
                     tooltip={t("nav.groups.stationManagement")}
                     isActive={isStationMgmtOpen}
                   >
                     <IconSettings2 />
                     <span>{t("nav.groups.stationManagement")}</span>
                     <IconChevronDown className="ms-auto transition-transform group-data-[state=open]/station-mgmt:rotate-180" />
                   </SidebarMenuButton>
                 </CollapsibleTrigger>
                 <CollapsibleContent>
                   <SidebarMenuSub>
                     {stationMgmtItems.map((item) => {
                       const resolved = resolvePath(item.to, item.params)
                       const label = t(item.labelKey)
                       const Icon = item.icon
                       return (
                         <SidebarMenuSubItem key={item.key}>
                           <SidebarMenuSubButton
                             asChild
                             isActive={isActive(item.key, resolved, pathname)}
                           >
                             <Link
                               to={item.to as LinkProps["to"]}
                               params={item.params as LinkProps["params"]}
                             >
                               <Icon />
                               <span>{label}</span>
                             </Link>
                           </SidebarMenuSubButton>
                         </SidebarMenuSubItem>
                       )
                     })}
                   </SidebarMenuSub>
                 </CollapsibleContent>
               </Collapsible>
             </SidebarMenuItem>
           </SidebarMenu>
         </SidebarGroupContent>
       </SidebarGroup>
     )
   }
   ```

4. The rest of the group loop handles all other groups exactly as before.

*Acceptance:* Owner sidebar shows "Station Management" with a chevron. Clicking
the trigger toggles expand/collapse. Navigating directly to a manage/* route
auto-expands the group (child shows as active). Custom User sidebar shows no
"Station Management" group.
