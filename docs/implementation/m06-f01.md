## Summary

Post-onboarding self-service management of a station's fuel prices for Owner + Manager, rendered as the second child of the Station Configuration hub. Replaces the `<UnderDevelopment />` placeholder. Users see a list of all the station's fuel types with their **current** price + when it became effective, can **set** a new price for any fuel type with an explicit effective date/time (closing the previous price atomically), and can drill into the **history** for any fuel type. Implements R01–R04. **Out of scope:** double-confirmation (M06-F02), notification fan-out (M06-F02), mid-shift handling (M06-F03), margins (M06-F04), customer special rates (M06-F05), promotions (M06-F06).

## Key model facts (from code review during planning)

- **`FuelPrices` is per-tenant** (`AppDbContext`, table `fuel_prices`). Columns: `Id`, `FuelTypeId` (Guid, cross-context plain ref to control-plane `FuelType`), `StationId`, `Price` (decimal), `EffectiveFrom` (DateTime UTC), `EffectiveTo` (DateTime? UTC, NULL = current/active).
- **R01 ("one active per fuel/station") is already enforced at the handler level.** `SetFuelPriceCommandHandler` calls `GetCurrentByStationAndFuelTypeAsync` and stamps the prior row's `EffectiveTo = newRow.EffectiveFrom` before inserting the new one. No DB constraint; matches the application-level pattern used by `StationShift` ([M04-F03-R01](../MODULES.md#m04-f03--open-shift)).
- Existing handlers / repo / controller / DTOs / Mediator wiring all exist:
  - Query: `GetFuelPricesByStationQueryHandler` — returns `List<FuelPricesDto>` for a station (all rows, current + historical), org-guarded.
  - Command: `SetFuelPriceCommandHandler` — org-guards, validates fuel-type exists, closes the active row, inserts the new one.
  - Repo: `IFuelPricesRepository` — `GetByStationIdAsync`, `GetCurrentByStationAndFuelTypeAsync`, `GetByIdAsync`, `AddAsync`, `Update`.
  - Controller: `[Authorize(Roles="Owner,Manager")] [Route("api/v1/stations/{stationId:guid}/fuel-prices")]` — `GET /` (list), `POST /` (set).
- Frontend lib already exists: `lib/api/stations/fuel-prices.ts` — `getFuelPricesByStation`, `setFuelPrice`, with matching DTO types.
- The setup-wizard pricing step (`station-setup/step2-prices.tsx`) is precedent for the per-fuel-type form pattern; reuse the visual semantics where it matches.

## Decisions (made during planning — confirm during implementation)

1. **"Set price" UX is one form, one effective-from field.** No "schedule" / "future-effective" pickers in v1 — just a `datetime-local` defaulted to *now*. The handler already accepts any `EffectiveFrom` (past, present, future) because it closes the prior row at that exact instant. We document the constraint in UI copy: "Prices effective from this moment forward, unless you change the date."
2. **Backdated prices are rejected.** New rule for this PR — validator returns 400 when `EffectiveFrom < UtcNow - 5 minutes` (the 5-minute window absorbs client clock skew). Reason: backdated prices would silently rewrite historical sales math when shifts close. The 5-minute slop comes from the `M04-F03` shift-open clock-skew handling.
3. **History view is a read-only Dialog**, not a separate route. Per-fuel-type, fetched on demand (the existing list endpoint already returns all rows, so no new endpoint — we filter client-side).
4. **Sellable badge is **not** duplicated here.** The "Sellable / Not yet sellable" state belongs to the Fuel Types panel (M08-F08). The Pricing page shows "No active price" inline for fuel types with no `EffectiveTo == null` row and a deep link to add one.
5. **DataTable is reused.** Columns: Fuel Type · Unit · Current Price · Effective Since · Last Set By · Actions. Faceted filters: Source (OMC / Custom — pulled from M08-F08), Has-active-price (yes / no). Mobile cards via `renderMobileCard`.
6. **Audit via Serilog now.** Add `_logger.LogInformation("AUDIT FuelPrice.Set …")` to `SetFuelPriceCommandHandler` mirroring the M08-F08 pattern. The persistent `AuditLog` table rides with `M01-F08` — R04 is satisfied, not deferred.
7. **Currency formatting.** `Intl.NumberFormat('en-PK', { style: 'currency', currency: 'PKR' })` from a tiny `lib/format/currency.ts` helper. Display as `Rs 285.50` (matching `step3-prices.tsx` precedent). Same helper will be reused by future modules (sales, reports, finance).
8. **"Effective Since" column shows relative time** ("2 days ago") with the absolute UTC instant in a tooltip. Format via a new `lib/format/relative-time.ts` helper (no new dep — uses `Intl.RelativeTimeFormat`).
