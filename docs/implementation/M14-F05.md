### Phase 1 — PhoneNumber unique index (ControlPlane migration)

- [x] In `server/FuelFlow.Infrastructure/Data/Configurations/ControlPlane/AppUserConfiguration.cs`
      add a unique index on `PhoneNumber`:
      ```csharp
      // Unique index — DB-level enforcement of phone uniqueness (M14-F05-R01)
      builder.HasIndex(u => u.PhoneNumber)
             .IsUnique()
             .HasFilter("\"PhoneNumber\" IS NOT NULL");
      ```
      A partial filter (`IS NOT NULL`) is required because Postgres treats UNIQUE
      indexes without a filter as allowing only one NULL — users who haven't
      provided a phone yet should not block each other.
- [x] Run `./server/db-migration-add.ps1 -Name AddPhoneUniqueIndex -Context ControlPlane`
      to generate the migration. Confirm the generated migration adds the index
      and does not touch other tables.
- [x] Run `./server/db-update.ps1 -Context ControlPlane` to apply.
- [x] **Acceptance:** Two `POST /auth/register` requests with the same phone
      number: first returns 200; second returns 400 (DB-constraint violation
      surfaced as a business-rule failure by `RegisterCommandHandler`'s
      catch path or the existing app-level check).
