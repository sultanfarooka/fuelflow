## Summary

M07-F07 is the cross-cutting authenticated layout — a persistent left sidebar
with role-aware navigation, a top bar (station switcher, user menu, language +
theme toggles), and an `<Outlet />` content area — that every per-module page
plugs into. It is the blocker for all module-specific pages (Finance, Reports,
Shifts, Inventory, Settings). The current `dashboard/route.tsx` chrome (a single
"Organization" link + user dropdown) is replaced by a reusable `AppShell`. To
let the shell be tested end-to-end and to give every future feature a home,
this item also ships placeholder ("Coming soon") route stubs for the planned
station-scoped module pages and a Settings route.

## Decisions (from planning interview)

1. **Unbuilt routes → create stub pages now.** Every sidebar link resolves to a
   real route rendering a shared `<ComingSoon />` page inside the shell. No 404s.
2. **Nav model → station switcher sets the active station.** Module nav links are
   station-scoped (`/dashboard/station/$activeStationId/...`). "All Stations"
   clears the active station and returns to the org consolidated dashboard
   (`/dashboard`), where only org-level items (Organization, Reports, Settings)
   show. Active station id is held in a small persisted Zustand UI store and kept
   in sync with the route `stationId` param.
3. **Sidebar → shadcn Sidebar primitive.** `npx shadcn@latest add sidebar` gives
   desktop collapse, mobile drawer (via the existing `Sheet`), state persistence,
   and the keyboard shortcut out of the box, using the `--sidebar*` tokens.

## Role → navigation mapping

Roles come from `UserInfo.roles` (values: `Owner`, `Manager`, `Nozzleman`,
`Accountant`, `Custom`). A nav item lists the roles allowed to see it; visibility
is the union across the user's roles. Mirrors the route→access table in
`fuel-flow-web/src/routes/CLAUDE.md`. **Frontend role checks are UX only** — the
API enforces access regardless.

| Nav item | Route | Scope | Owner | Manager | Nozzleman | Accountant | Custom |
|---|---|---|:-:|:-:|:-:|:-:|:-:|
| Organization | `/dashboard` | org | ✓ | ✓ | | ✓ | ✓ |
| Station overview | `/dashboard/station/$id` | station | ✓ | ✓ | | | |
| Shifts | `/dashboard/station/$id/shifts` | station | ✓ | ✓ | ✓ | | |
| Inventory | `/dashboard/station/$id/inventory` | station | ✓ | ✓ | | | |
| Finance | `/dashboard/station/$id/finance` | station | ✓ | ✓ | | ✓ | |
| Reports | `/dashboard/station/$id/reports` | station | ✓ | ✓ | | ✓ | |
| Setup | `/dashboard/station/$id/setup` | station | ✓ | ✓ | | | |
| Settings | `/settings` | org | ✓ | | | | |

- **AC1 check:** a Nozzleman's only allowed item is **Shifts** — no Finance,
  Reports, or Settings. ✓
- Station-scoped items render only when an active station is selected; under
  "All Stations" only org-scoped items (Organization, Settings) show.

## Implementation notes

- **shadcn CLI overwrite reverted.** `npx shadcn add sidebar --overwrite` also
  rewrote 4 existing owned primitives (`button`, `separator`, `sheet`,
  `tooltip`), swapping the project's RTL **logical** utilities for physical
  ones and dropping `button.tsx`'s intentional `eslint-disable`. Those 4 were
  reverted with `git checkout`; only the genuinely-new files were kept
  (`ui/sidebar.tsx`, `ui/skeleton.tsx`, `hooks/use-mobile.ts`).
- **Vendored sidebar = sanctioned exception.** `ui/sidebar.tsx` keeps the
  preset's physical `left/right` utilities keyed to its dual-side system; we
  anchor a single sidebar at inline-start and documented the exception in
  `components/CLAUDE.md`. All `layout/*` components we wrote use logical
  utilities (audit greps pass).
- **Breakpoint aligned to 640.** R04/AC3 require the drawer < 640px. Both the
  `use-mobile` hook (`MOBILE_BREAKPOINT = 640`) **and** the sidebar's show/hide
  gate (`md:` → `sm:`, 2 spots) were moved to 640 so the desktop sidebar can't
  vanish in a 640–768 gap.
- **Testing approach.** The frontend has no unit-test runner installed (no
  vitest/jest/RTL); the repo's established frontend test mechanism is Playwright
  e2e specs (`e2e-tests/*.spec.ts`). Rather than introduce a new test stack for
  one feature, R01–R05 ACs are verified by the e2e spec produced in Step 4.5
  (`e2e-tests/M07-F07.spec.ts`). `getNavItems` is kept a pure function so its
  role/scope rules stay cleanly verifiable (and unit-testable later if a runner
  lands).
- **Icons.** New shell code uses Tabler (`@tabler/icons-react`, the preset
  default). `IconFuel` doesn't exist in Tabler — brand mark uses `IconGasStation`.
- **Active-station sync** lives in `AppShell` via `useParams({ strict: false })`
  so deep-linking to a station route updates the switcher + station-scoped nav.
