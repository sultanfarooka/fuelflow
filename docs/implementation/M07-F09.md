## Summary

Migrate `fuel-flow-web` from Tailwind 3.4 / shadcn 1.x to Tailwind 4 /
shadcn 2.x (prerequisite ŌĆö the `b3lVLqquH` preset is generated as T4-only

minimum-viable RTL infrastructure (native Tailwind logical utilities +
`document.documentElement.dir` toggle wired into a new language switcher),
migrate the 4 known token-violating screens (success states using
`green-*`) to a new `--success` token, sweep all 29 shipped
screens/components for compliance, and document the going-forward
standard in `fuel-flow-web/src/components/CLAUDE.md`. One PR; one
MODULES.md status flip.

The frontend is **already mostly token-clean today** ŌĆö the planning
audit found zero inline `style={`, zero hex literals in components, no
`window.confirm`, no rogue brand colors. The token/utility migration
itself is narrow: 4 files for success tokens, ~25 files for
directional-utility swaps, and the auth layout for missing toggles. The
**bulk of the new work** introduced by the scope expansion is the
Tailwind 4 upgrade (CSS-first config, `@import "tailwindcss"`, Vite
plugin swap, primitive class-name compatibility, oklch token format)
which lands in Phase 1.

## Key technical decisions

- **Tailwind 4 migration (new ŌĆö added during Phase 1 discovery).** The
  `b3lVLqquH` preset is generated by the modern shadcn theme builder and
  produces **Tailwind 4 / shadcn 2.x** output: `oklch(...)` tokens,
  `@import "tailwindcss"`, `@import "shadcn/tailwind.css"`,
  `@import "tw-animate-css"`, `radix-ui` umbrella imports, and primitives
  using T4-only utilities (`rounded-4xl`, `ring-3`, `not-aria-[ŌĆ”]:`,
  `has-data-[ŌĆ”]:`). The project is on Tailwind 3.4 + shadcn 1.x; the
  preset cannot be applied cleanly without first migrating. Migration
  uses the official `npx @tailwindcss/upgrade@latest` tool to rewrite
  `tailwind.config.js` ŌåÆ `@theme` in `index.css`, swap PostCSS for the
  `@tailwindcss/vite` plugin, replace `@tailwind base/components/utilities`
  with `@import "tailwindcss"`, and audit class-name compatibility.
- **Preset:** `npx shadcn@latest apply b3lVLqquH -y` (R01), applied
  **after** the Tailwind 4 migration completes. Will rewrite the 11
  existing primitives (radix-mira style), replace tokens with oklch
  scale, switch icon library to Tabler (Lucide stays installed for
  pre-existing imports), and add Inter Variable as the heading/sans
  font. Adds `react-hook-form` as a dep ŌĆö **removed immediately
  post-apply** (see Form-primitive deviation).
- **Primitive scope (R02):** install **Dialog, Sheet, Table, Tabs,
  Tooltip** ŌĆö completes the R02 list minus Form (see deviation). Already
  shipped: Button, Input, Select, Card, Badge, Alert, Label, Separator,
  Dropdown-Menu, Field, Sonner.
- **shadcn `Form` primitive ŌĆö INTENTIONALLY OMITTED.** Shadcn's `Form`
  wraps `react-hook-form` via `useFormContext`; the project uses
  TanStack Form via the existing `Field` + `FormTextField` stack. Adding
  shadcn `Form` would introduce a conflicting forms story and an unused
  `react-hook-form` dependency. The R02 row in the shipping PR is
  annotated with this deviation.
- **Success-state token:** add paired light/dark `--success` tokens in
  `index.css` next to `--destructive`, **in the preset's oklch format**
  (the project moves to oklch as part of the Tailwind 4 migration in
  Phase 1a). Suggested starting values: light
  `--success: oklch(0.6 0.18 145);` dark
  `--success: oklch(0.7 0.16 145);`. Tune during implementation. With
  Tailwind 4's `@theme { --color-success: var(--success); }` token
  binding, the 4 green-using files migrate in Phase 5 to `bg-success/10
  text-success` (no arbitrary value syntax needed).
- **RTL utilities:** native Tailwind 3.4 logical utilities
  (`ms-`/`me-`/`ps-`/`pe-`/`start-`/`end-`/`text-start`/`text-end`/
  `border-s`/`border-e`/`rounded-s-*`/`rounded-e-*`). **No
  `tailwindcss-rtl` dependency.** `components.json` flip to
  `"rtl": true` makes future `shadcn add` runs emit logical utilities
  automatically.
- **`components.json`:** flip `"rtl": true`. Keep `style: "new-york"`,
  `baseColor: "neutral"`, `cssVariables: true`.
- **Dark-mode infra:** already correct ŌĆö `ThemeProvider` in `main.tsx`
  wraps the whole tree. The only gap is `<ModeToggle />` not being
  mounted on the auth layout; fixed in Phase 3.
- **RTL trigger:**
  `i18n.on('languageChanged', lng => document.documentElement.dir = lng === 'ur' ? 'rtl' : 'ltr')`
  wired in `main.tsx`. Initial `dir` set from persisted language on
  bootstrap so first paint is correct.

false-positives (rewrote shadcn `variant="outline"` prop values to
`"outline-solid"` in `station-setup/{step3-tanks,step4-dip-charts}.tsx`
and `routes/index.tsx`). Two were **pre-existing on `main`**: the
`@/lib/api/auth` barrel was missing `ForgotPasswordResponse` re-export
(used by `auth/forgot-password-form.tsx`), and both `auth/login-form.tsx`
and `auth/forgot-password-form.tsx` passed `inputMode="text"` which is
not assignable to `FormTextField`'s restricted prop type. All five
fixed in commit `9ca4c28`. The `FormTextField`'s `FieldApi.errors`
type was also tightened from `unknown[]` to
`({ message?: string } | undefined)[]` to match the new shadcn
`FieldError` primitive's signature.
